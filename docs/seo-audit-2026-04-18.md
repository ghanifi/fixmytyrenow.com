# SEO Audit Report — fixmytyrenow.com

**Date:** 2026-04-18
**Site:** FixMyTyreNow — Mobile Tyre Fitting London
**Pages Analysed:** 339 (32 borough pages × 5 service combos + 10 core pages, sampled in depth)
**Overall Score:** **78/100** — Good foundation with specific fixes needed

---

## Critical Issues (must fix)

- [ ] **Franchise FAQPage schema contradicts page content** — `franchise-registration/index.html:31`
  The JSON-LD FAQPage schema says *"No prior experience required. Full operational training is provided"* but the visible page HTML says *"Yes — we require franchise partners to be qualified tyre technicians."* Google can penalise for structured data that misleads users vs page content. The schema block needs to be rewritten to match the HTML FAQ answers exactly.

- [ ] **Meta description on franchise page is 191 chars** — `franchise-registration/index.html:10`
  Google truncates at ~160 chars. The current description will be cut mid-sentence in SERPs. Trim to ≤160 chars while keeping the key value proposition ("confirmed deposit-paid jobs" + "£25/month").

---

## Warnings (should fix)

- [ ] **All 339 pages share the same OG image** (`hero.jpg`)
  Every page — from the homepage to `areas/wandsworth/puncture-repair/` — outputs the same social sharing image. Area pages and service pages shared on social will look identical. Add page-type-specific OG images: one for area pages, one for service pages, one for the franchise page.

- [ ] **Area page titles exceed 60 chars** — e.g. `areas/barnet/index.html:9`
  "Mobile Tyre Fitting Barnet | 24/7, 20-Min Arrival | FixMyTyreNow" = **64 chars**. Google clips at ~60. Likely affects all 32 borough pages. Shorten pattern to e.g. `"Mobile Tyre Fitting Barnet | 20-Min Arrival | FixMyTyreNow"` (58 chars).

- [ ] **Franchise page title is 65 chars** — `franchise-registration/index.html:9`
  "Mobile Tyre Franchise UK | Join FixMyTyreNow | Earn Per Postcode" — 2 chars over. Drop "Earn Per Postcode" and rephrase the third pipe segment.

- [ ] **Services hub title is only 45 chars** — `services/index.html:9`
  "Our Mobile Tyre Services | FixMyTyreNow London" is short. Adding "– Emergency, Fitting & Repair" would reach ~57 chars and improve keyword coverage.

- [ ] **Several meta descriptions are under 150 chars**
  - Homepage: ~135 chars (`index.html:10`) — add one more value prop (e.g. "Book in 60 seconds")
  - Services hub: ~134 chars (`services/index.html:10`) — undersells the page
  - Barnet (representative of all area pages): ~142 chars (`areas/barnet/index.html:10`) — similar pattern across all 32 boroughs

- [ ] **About page has no FAQPage schema** — `about/index.html`
  The homepage, all area pages, and the franchise page all have FAQPage structured data. The About page has none. An FAQ like *"When were you founded?" / "How many boroughs do you cover?"* would add a rich result opportunity with minimal effort.

- [ ] **`AutomotiveBusiness` on services page has wrong `url`** — `services/index.html:26`
  The `@id` and `url` fields in the AutomotiveBusiness block point to `https://fixmytyrenow.com/services/` instead of the canonical business URL `https://fixmytyrenow.com`. This can dilute entity consolidation in Google's Knowledge Graph.

- [ ] **Sitemap has no `<priority>` values** — `sitemap.xml`
  339 URLs all omit `<priority>`. While Google has de-emphasised this, adding `1.0` for the homepage, `0.9` for service/area hub pages, `0.8` for borough pages, and `0.6` for combo pages helps signal crawl budget allocation.

- [ ] **CSS is render-blocking** — `index.html:29-30`
  Both `fonts.css` and `main.css@ver=1.3.4.css` are loaded as standard `<link>` tags with no `media` attribute or preload. Consider `<link rel="preload" as="style">` + `onload` swap for non-critical CSS, or inline the critical above-fold CSS.

## Opportunities (nice to have)

- [ ] **No `<priority>` or `<changefreq>` on combo pages**
  The 160 area+service combo pages (e.g. `/areas/barnet/puncture-repair/`) likely have lower crawl frequency needs than borough hub pages. Differentiated values would help.

- [ ] **`og:type` is `"website"` on all pages**
  Service and area pages would benefit from `og:type: "service"` or a more specific type to improve social sharing appearance.

- [ ] **`WebSite` schema only on homepage**
  The `WebSite` JSON-LD with `SearchAction` (sitelinks search box) is present on the homepage. Ensuring `SearchAction` is included would enable sitelinks search in Google SERPs.

- [ ] **No `Person` or `founder` schema on About page**
  Adding founder details to the About page's AutomotiveBusiness schema would strengthen E-E-A-T signals, which matter for local service businesses.

- [ ] **Reviews page not audited** — `/reviews/` should have `AggregateRating` or `Review` schema to support star ratings in SERPs.
