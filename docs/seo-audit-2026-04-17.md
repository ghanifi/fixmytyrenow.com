## KRİTİK SORUNLAR (Hemen Düzelt)

### ✅ 1. Cloudflare tüm AI tarayıcılarını engelliyor
**Kategori:** GEO / AI Görünürlüğü

Cloudflare'in sunduğu canlı `robots.txt`, yerel dosyayı tamamen geçersiz kılarak şunları engelliyor:
`GPTBot`, `ClaudeBot`, `Google-Extended`, `Applebot-Extended`, `CCBot`, `Amazonbot`, `meta-externalagent`

Yerel dosya doğru. Sorun tek bir Cloudflare dashboard ayarı.

**Düzeltme:**
- Cloudflare Dashboard → Security → Bots → **"Block AI Scrapers and Crawlers"** kapat
- `robots.txt` dosyasını güncelle (AI arama botlarına izin ver, eğitim botlarını engelle)

```
User-agent: GPTBot
Allow: /

User-agent: OAI-SearchBot
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: Google-Extended
Allow: /

User-agent: CCBot
Disallow: /

User-agent: *
Allow: /
Disallow: /booking/
Disallow: /franchise-registration/
Disallow: /wp-json/

Sitemap: https://fixmytyrenow.com/sitemap.xml
```

- [ ] Tamamlandı

### ✅ 2. 160 combo sayfasında bozuk CTA anchor
**Kategori:** Dönüşüm / İçerik

Her combo sayfasında `<a href="./#booking-form">Book Now</a>` var ama form bölümünün ID'si `id="book"`. Sayfanın ortasındaki CTA düğmesi hiçbir yere gitmiyor.

**Düzeltme:** `./#booking-form` → `./#book` (160 sayfada Python scriptiyle)

- [ ] Tamamlandı

---

### ✅ 3. Favicon hiçbir sayfada yok
**Kategori:** Teknik / Görsel

208 sayfanın hiçbirinde `favicon.ico`, `<link rel="icon">`, `apple-touch-icon` veya `site.webmanifest` yok. Tarayıcı sekmelerinde boş ikon görünüyor.

**Düzeltme:**
```html
<link rel="icon" type="image/x-icon" href="/favicon.ico">
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
```

- [ ] Tamamlandı

---

### ✅ 4. Hero görseli preload yok — LCP etkisi
**Kategori:** Performans / Core Web Vitals

Hero `<img loading="eager">` için `<link rel="preload">` ipucu yok. Tarayıcı HTML tamamen parse edilene kadar görseli indirmeye başlayamıyor. Doğrudan LCP cezası.

**Düzeltme:** Her sayfanın `<head>` bölümüne ekle:
```html
<link rel="preload" as="image" href="/wp-content/themes/FixMyTyreNow_edit/assets/images/hero.jpg" fetchpriority="high">
```

- [ ] Tamamlandı

---

### ✅ 5. WordPress REST API URL her sayfanın JS'inde açık
**Kategori:** Güvenlik / Teknik

Her sayfada bu var:
```javascript
var fmtnConfig = {"restUrl":"https://fixmytyrenow.com/wp-json/fmtn/v1/","phone":"07340645595","deposit":"10.00"};
```
Canlı WordPress backend API URL'si 208 statik sayfada herkese açık. WP kurulumu güvenli değilse saldırı yüzeyi.

**Düzeltme:**
- WordPress backend'de `/wp-json/` endpoint'lerinde kimlik doğrulama zorunlu hale getir
- Rate limiting ekle, XML-RPC'yi kapat, kullanıcı listelemeyi engelle
- `robots.txt`'e `Disallow: /wp-json/` ekle

- [ ] Tamamlandı

## YÜKSEK ÖNCELİK (1 Hafta İçinde)

### ✅ 6. FAQPage schema hiçbir sayfada yok
**Etki:** Google AI Overviews, Bing Copilot, Perplexity direkt cevaplar

FAQ içeriği HTML'de mevcut: ana sayfada 7 soru, 32 borough sayfasında 5'er soru (`<details>/<summary>` ile) — ama hiçbir sayfada `FAQPage` JSON-LD yok. Not: Google, Ağustos 2023'ten itibaren ticari sitelerde FAQ rich results'ı kısıtladı ama FAQPage schema, Google AI Overviews ve LLM atıf çıkarımı için #1 tetikleyici olmaya devam ediyor.

**Düzeltme:** 33 sayfaya (ana sayfa + 32 borough) FAQPage schema ekleyen Python scripti.

- [ ] Tamamlandı

---

### ✅ 7. openingHours schema ile sayfa içeriği çelişiyor
**Etki:** Google Knowledge Panel, AI cevap doğruluğu

Schema: `Mo-Su 07:00-21:00`
Sayfada görünen: Acil Servis 24/7, Programlı 06:00–22:00, Telefon 07:00–23:00

**Düzeltme:**
```json
"openingHoursSpecification": [
  {
    "@type": "OpeningHoursSpecification",
    "dayOfWeek": ["Monday","Tuesday","Wednesday","Thursday","Friday","Saturday","Sunday"],
    "opens": "06:00",
    "closes": "22:00",
    "name": "Scheduled Service"
  }
]
```
Ve description alanına 24/7 acil servis notunu ekle.

- [ ] Tamamlandı

---

### ✅ 8. Review schema'da `datePublished` eksik
**Etki:** Rich result uygunluğu, schema doğrulama

Tüm combo sayfalarındaki standalone `Review` bloklarında `datePublished` yok. Ayrıca aynı review (Sarah M.) hem parent borough hem de tüm child combo sayfalarında kullanılıyor — bu misleading structured data uyarısına yol açabilir.

**Düzeltme:**
- Her Review bloğuna `"datePublished": "YYYY-MM-DD"` ekle
- Her sayfada farklı, özgün bir review kullan

- [ ] Tamamlandı

---

### ✅ 9. `@id` entity linking hiçbir sayfada yok
**Etki:** Knowledge Graph entity bağlantısı

Hiçbir `AutomotiveBusiness` bloğunda `"@id": "https://fixmytyrenow.com/#business"` yok. Bunu olmadan Google, 208 sayfadaki entity'yi tutarlı bir Knowledge Graph node'una bağlayamıyor.

**Düzeltme:** Tüm `AutomotiveBusiness` bloklarına Python scriptiyle ekle:
```json
"@id": "https://fixmytyrenow.com/#business"
```

- [ ] Tamamlandı

---

### ✅ 10. 208 sayfanın hiçbirinde isim belirtilen kişi yok
**Etki:** E-E-A-T — Deneyim ve Uzmanlık faktörleri

Hiçbir teknisyen, kurucu veya çalışan adı yok — About sayfasında bile. Yol güvenliğiyle ilgili bir hizmet için Google QRG değerlendiricileri insan hesap verebilirliği arıyor.

**Düzeltme:**
- About sayfasına en az bir kişinin adını, geçmişini ve fotoğrafını ekle
- `Person` schema markup ekle

- [ ] Tamamlandı

### ✅ 11. 160 combo sayfası %80 özdeş içerik yapısı
**Etki:** Thin content riski, indeksleme kalitesi

`borough-process` (5-adım liste), `borough-benefits` (5-madde liste) ve `borough-why-us` bölümleri 160 sayfada aynı. Sadece giriş paragrafı benzersiz. Bu, Mart 2024 Helpful Content güncellemesi sonrası tüm sitenin sıralama potansiyelini bastırabilir.

**Kanıt:** Barnet/puncture-repair FAQ'ı "30-60 dakika" diyor, aynı borough'un ana sayfası "20 dakika" diyor.

**Düzeltme:**
- Her combo sayfasına o borough+hizmet kombinasyonuna özgü içerik ekle
- Tüm FAQ cevaplarını 20 dakika varış iddiasıyla tutarlı hale getir
- `borough-process` ve `borough-benefits` bölümlerini kaldır veya servis+lokasyon özgü yap

- [ ] Tamamlandı

### ✅ 12. Servis sayfaları kritik derecede az içerik
**Etki:** E-E-A-T Uzmanlık sinyalleri

`/services/standard-tyre-fitting/` ~230 kelime içeriyor — servis sayfaları için gereken 800 kelimenin çok altında. Marka adı yok, montaj süreci açıklanmıyor, teknisyen yeterlilikleri yok.

**Düzeltme:** Her servis sayfasını 800+ kelimeye çıkar:
- Adım adım ne yapıldığı
- Hangi marka lastikler stokta
- Fiyatlandırma yapısı
- Yasal asgari sınır bilgileri
- Mevsimsel tavsiyeler

- [ ] Tamamlandı

### ✅ 13. Hero görseli `width`/`height` attribute yok — CLS
**Etki:** Core Web Vitals CLS skoru

Ana sayfadaki hero `<img>` için boyut belirtilmemiş. Görsel yüklenene kadar yer ayrılamıyor, layout kayması oluyor.

**Düzeltme:**
```html
<img src="..." alt="..." loading="eager" width="800" height="600">
```
(Görselin gerçek piksel boyutlarını kullan)

`main.js`, `revolut-embed.js`, `booking.js` dosyaları `<body>` sonunda ama `defer` attribute'suz.

**Düzeltme:** 208 sayfada tüm script tag'lerine `defer` ekle.
